# Paket I — Metode Pembayaran (dashboard), Bonus kategori detail, Alasan review

> **For agentic workers:** dieksekusi via Workflow ultracode (3 scout paralel → 3 implementer TDD sekuensial → suite + 2 reviewer adversarial + ronde perbaikan), lalu verifikasi browser inline.

**Goal (permintaan klien, 5 screenshot):** (I1) kartu dashboard "penggunaan bank" — DP per Bank/QRIS/E-wallet & WD per Bank/Nexuspay/QRIS, acuan **Bank Title panel** ("buat QR pake acuan bank tittle bank kita"); (I2) kategori Rekonsiliasi Bonus membaca nama promo asli ("BONUS ROLLINGAN SLOT 0.5% DAILY") + filter kategori + summary value; (I3) aksi Setujui/Tinjau membawa **alasan** seperti modal Koreksi FR/Bracket.

## Keputusan desain

- **I1** `web/channels.py` murni; sumber = queryset `panel_sum` yang sama (baris panel ter-konsumsi batch terakhir) → total breakdown WAJIB == strip ringkasan. Klasifikasi case-insensitive dari Bank Title raw panel: DP urutan `QR` → e-wallet (DANA/OVO/GOPAY/SHOPEEPAY/LINKAJA; "DANAMON" dikecualikan dari "DANA") → Bank → Lainnya; WD urutan `NEXUSPAY/NXPAY` → `QR` → Bank → Lainnya. "NXPAY DEPOSIT QR" = QRIS (klien menandai QR-nya); "QRIS NEXUSPAY" WD = Nexuspay. Lainnya disembunyikan saat 0. Jebakan locale: `{{ pct }}` ter-render "68,7" → width bar pakai `floatformat:"0u"`.
- **I2** display/agregasi query-time SAJA — pairing (username+nominal+tanggal, greedy 1:1) tidak berubah; filter diterapkan pada hasil yang SUDAH berpasangan. Ekstraksi detail: `"<Kategori>: <DETAIL> - [tiket] - user"` → DETAIL; fallback kategori lama; sisi bracket tetap nama kategorinya (ruang nama panel vs bracket terpisah — pasangan lintas-nama tampil 2 baris ringkasan, kolom nilai justru membuatnya terlihat). Ringkasan per kategori + kolom Nilai (Rp) per bucket + baris TOTAL; dropdown mempertahankan param lain; filter aktif di luar rentang tetap muncul sebagai opsi terpilih (anti "menyaru Semua kategori").
- **I3** daftar alasan **reuse konstanta `FRKoreksi`** (impor, bukan duplikat). `ReviewAction.alasan` TANPA `choices=` di model (validasi di view; layering reconciliation↛web; ganti daftar tanpa migrasi) — migrasi reconciliation 0006 additive. Modal seragam Koreksi: single (fragmen htmx server-rendered per baris) + bulk (`_bulk_alasan_modal.html`) di run detail DAN Area Pengecekan; "Lainnya" memunculkan catatan; POST tanpa alasan = perilaku lama (kompat mundur, tes lama tak diubah).
